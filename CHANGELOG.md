# Changelog

All notable changes to this package will be documented in this file.

The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

## [0.2.0] - 2026-02-07

### Changed

- Renamed specification domain directories to semantic names and updated all references.
- Clarified L0/L1 layer vocabulary and removed ambiguous Lxx terminology.
- Regenerated the spec manifest to align with the new layout.

### Added

- Release workflow steps for tests/lint/build when present, plus GitHub Releases on tag push.

## [0.1.0] - 2026-02-06

### Added

- Initial publication of the specification documents under `specifications/`.
- `spec.manifest.json` to provide stable spec-id based lookup for consumers.
